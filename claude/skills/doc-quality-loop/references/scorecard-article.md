# スコアカード v1（article）

記事ドラフトの品質を判定するための評価基準。
Generator が出力したドラフトに対し、Reviewer（初心者・批判的読者）がこのスコアカードで判定する。

## MUST（全通過必須）

すべて PASS でなければ収束しない。1つでも FAIL なら改善ループを継続する。

| ID | 項目 | 判定基準 | FAIL の典型例 |
|---|---|---|---|
| M1 | 読者価値3チェック通過 | ①行動変化の具体性: 読了後に読者が「何をできるようになるか」が明確。②情報希少性: 公式ドキュメントのコピーではなく独自の切り口がある。③アクセシビリティ: 前提知識が明示され、読者が自力で追える | 「〜について解説します」で終わり行動変化なし。公式ドキュメントの和訳。前提知識の説明がゼロ |
| M2 | 各セクションが独立して読める | セクション内で前方参照（「後述する〜」「次の章で〜」）がない。各セクションが自己完結している | 「これについては後ほど詳しく説明します」が繰り返される |
| M3 | 導入部で読者を決めつけていない | 「〜した人は少ない」「〜を知らない人が多い」「〜を読んだことがある人は少ないはず」が不在。自分の発見・体験から入っている | 「〜した人は少ないのではないでしょうか」で始まる |
| M4 | 具体的アクション含む | コマンド、手順、コード例のいずれかが1つ以上含まれている。読者が手を動かせる内容がある | 概念の説明だけで実行可能な手順がゼロ |
| M5 | フォーマット準拠 | Zenn: h1不使用 + 画像パス絶対パス + `## `以降で始まる。Qiita: 該当フォーマットルール準拠 | `# タイトル` が本文中にある。画像パスが `../images/` の相対パス |

## WANT（改善推奨）

MUST を全通過した上で、品質向上のために判定する。

| ID | 項目 | 判定基準 | YES の例 |
|---|---|---|---|
| W1 | 1文1メッセージ | 各記述が1文に1つのメッセージのみ含む。接続詞で複数要因を繋いでいない | 短く分割された箇条書き。「〜し、〜で、〜が」の三重接続がない |
| W2 | AI的定型表現の排除 | 「〜を推進する」「〜を強化する」「〜を図る」「〜と言えるでしょう」等の抽象動詞・定型文がない | 具体的な動詞（「書く」「実行する」「確認する」）が使われている |
| W3 | トレードオフの提示 | メリットだけでなくデメリット・制約・向かないケースが言及されている | 「ただし、〇〇の場合はこのアプローチは向かない」が含まれる |
| W4 | 再現可能性 | コード例・手順に環境前提（OS、バージョン、依存パッケージ）が明記されている | 「Node.js 18以上、macOS/Linux」等の前提が書かれている |
| W5 | 導入が体験から始まる | 冒頭が自分の発見・体験・具体的なフックから入っている | 「〇〇というコマンドを偶然見つけた」「〇〇を試していて気づいた」 |

## 収束条件

```
IF MUST 全項目 PASS:
  IF ループ回数 >= 2 OR WANT の YES が 3/5 以上:
    → 完了。最終ドラフトを出力
  ELSE:
    → WANT 改善のためもう1ループ
ELSE:
  IF ループ回数 >= 5:
    → 打ち切り。残存課題レポートを人間に提示
  ELSE:
    → MUST FAIL 項目の改善指示を出して次ループ
```

## 判定出力フォーマット

Reviewer は以下の形式で判定を出力する:

```markdown
### MUST 判定

| ID | 判定 | 根拠 | 該当箇所 |
|---|---|---|---|
| M1 | PASS/FAIL | （具体的な根拠） | （ドラフト内の該当箇所を引用） |
| M2 | PASS/FAIL | （具体的な根拠） | （該当箇所） |
| M3 | PASS/FAIL | （具体的な根拠） | （該当箇所） |
| M4 | PASS/FAIL | （具体的な根拠） | （該当箇所） |
| M5 | PASS/FAIL | （具体的な根拠） | （該当箇所） |

### WANT 判定

| ID | 判定 | 根拠 | 該当箇所 |
|---|---|---|---|
| W1 | YES/NO | （具体的な根拠） | （該当箇所） |
| W2 | YES/NO | （具体的な根拠） | （該当箇所） |
| W3 | YES/NO | （具体的な根拠） | （該当箇所） |
| W4 | YES/NO | （具体的な根拠） | （該当箇所） |
| W5 | YES/NO | （具体的な根拠） | （該当箇所） |
```
