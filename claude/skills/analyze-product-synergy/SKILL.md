# Analyze Product Synergy

入力されたアイデアに対して、自社プロダクト群とのシナジー（アップセル・クロスセル・機能連携）を分析する。

## 入力

- `$ARGUMENTS`: 分析対象のファイルパス、またはアイデアの概要テキスト
- ファイルパスの場合は内容を読み取る

## 出力先

- 単独実行: 入力ファイルと同じディレクトリに `[アイデア名]-product-synergy.md`
- `/idea-feedback` から呼び出し: 親スキルが指定するディレクトリに出力
  - ソリューション仮説: `ideas/solutions/[アイデア名]/[アイデア名]-product-synergy.md`

## 自社プロダクト情報

分析前に必ず `resources/In-house-products/` 配下の全ファイルを読み込み、最新のプロダクト情報を把握する:

- `MCE.md` - モチベーションクラウド（エンゲージメントサーベイ）
- `MCS.md` - モチベーションクラウドストレッチ（組織改善支援）
- `MCR.md` - モチベーションクラウドロールディベロプメント（マネジャー育成）
- `MCM.md` - マネジメントAI / モチマネくん（マネジャー向けAIアシスタント）
- `BRIDGE_pit.md` - BRIDGE採用拡張（採用候補者向け適性検査）
- `MCB.md` - モチベーションクラウドベーシック（中堅中小向け組織改善サービス）
- `Unipos.md` - Unipos（全社参加型カルチャープラットフォーム）
- `ロボパット.md` - ロボパットDX（RPAツール）

## Workflow

### Phase 1: アイデアの要素分解

アイデアを以下の観点で分解する:
- **ターゲット**: 誰向けか（経営/人事、マネジャー、メンバー、採用候補者）
- **提供価値**: 何を解決するか
- **データ**: どんなデータを生成・活用するか
- **タッチポイント**: どの業務シーンで使われるか
- **導入タイミング**: 顧客のどのフェーズで導入されるか

### Phase 2: プロダクト別シナジー分析

各自社プロダクトとの接点を分析する:

#### 分析観点
1. **アップセル可能性**: このアイデアの導入が既存プロダクトの追加導入につながるか
2. **クロスセル可能性**: 既存プロダクト利用顧客にこのアイデアを追加提案できるか
3. **データ連携**: プロダクト間でデータを共有・活用できるか
4. **機能補完**: 既存プロダクトの弱み・不足を補えるか
5. **顧客体験の一貫性**: 組み合わせることで顧客体験が向上するか

### Phase 3: シナジーパターンの特定

以下のパターンに分類する:

- **バンドル型**: セット販売で提供価値が増幅する
- **データフロー型**: あるプロダクトのアウトプットが別のインプットになる
- **ジャーニー型**: 顧客のライフサイクルの異なるフェーズをカバーする
- **同一ターゲット型**: 同じ意思決定者に複数価値を提供できる

### Phase 4: 実現性と優先度の評価

- 技術的な連携難易度
- 営業・マーケティング上の自然さ（ストーリーが通るか）
- 顧客にとっての追加コスト対効果
- 既存プロダクトへの影響（ポジティブ/ネガティブ）

## 注意事項

- 無理に全プロダクトとの連携を見出す必要はない。シナジーがない場合は正直に「低い」と評価する
- 顧客目線で「自然な提案ストーリー」が成り立つかを重視する
- 既存プロダクトの情報は `resources/In-house-products/` から必ず最新を読み込む
- カニバリゼーション（共食い）のリスクも正直に記載する
- 連携の実現性は技術面だけでなく、営業・オペレーション面も考慮する
