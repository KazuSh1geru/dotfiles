# 戦略カスケード 6ステップ

新規プロジェクトや施策の企画時に、上流から下流へ論理的に整合性を保ちながら設計するためのフレームワーク。
`principles.md` の戦略カスケードを実務で適用する際の詳細手順。

## 原則：戦略のカスケードダウン

> **上流（Objective/Who/What）が間違っていると、下流（How）で挽回することは不可能**

```
Objective（なぜやるか・達成すべき目的）
    ↓ なぜこのWHOなのか？
Who（誰に届けるか）
    ↓ この人たちにとっての価値は？
What（何を届けるか）
    ↓ どう実現するか？
How（どうやって届けるか）
```

## Step 1: 背景とObjectiveを分離する

| 項目 | 問い | 例 |
|---|---|---|
| **背景（自社視点）** | なぜ自分たちがこれをやるのか？ | 「AI前提のワークフロー再設計」事例を増やしたい |
| **Objective（顧客視点）** | 顧客にとって何が達成されるのか？ | メンバーの行動改善、マネージャーのFBコスト削減 |

**注意**: Objectiveを「自社視点」で書くと、Whoとの接続が曖昧になる。

## Step 2: Whoを構造化する

### ターゲットの二層構造

| 層 | 定義 |
|---|---|
| **戦略ターゲット** | 事業規模を担保する広い層 |
| **コアターゲット** | マーケティング予算・メッセージを突き刺す具体的な層 |

### コアターゲット選定の6視点

1. **ペネトレーション**: まだ使っていない層
2. **ロイヤルティ**: 自社ばかり使うようにできる層
3. **コンサンプション**: 消費量・単価を増やせる層
4. **システム**: 関連商品も併せて買う層
5. **パーチェス・サイクル**: 頻度を上げられる層
6. **ブランド・スイッチ**: 競合から奪える層

### インサイトの言語化（必須）

| 種類 | 定義 | 問い |
|---|---|---|
| **マインド・オープニング** | 頭で理解している課題 | 「◯◯がわからない」「◯◯に時間がかかる」 |
| **ハート・オープニング** | 口に出さない本音 | 「本当は◯◯したくない」「◯◯が怖い」 |

> インサイトが空欄だと「なぜこのWhatが刺さるのか」の説得力がなくなる

## Step 3: Whatを定義する

### Whatは「商品」ではなく「便益」

- NG: 「FBが届く仕組み」（機能）
- OK: 「自分では気づけない盲点を可視化する」（便益）

### Output と Outcome を分離する

| 項目 | 定義 | 例 |
|---|---|---|
| **MVP（Output）** | まず作るもの | FBが自動で届く仕組み |
| **成功判断（Outcome）** | それによって起きる変化 | FBを見て改善するサイクルが習慣化される |

## Step 4: Howを判断ポイントに照らして設計する

### 判断ポイントを先に定義する

| 判断ポイント | 基準に照らした問い |
|---|---|
| ◯◯の実現 | ◯◯を達成できるか？ |
| △△の低さ | ユーザーの手間を最小化できるか？ |
| □□設計 | 具体的な行動変容につながるか？ |

### シナリオを判断ポイントで評価する

| シナリオ | 判断ポイント① | 判断ポイント② | 採用可否 |
|---|---|---|---|
| A案 | ◎ | △ | △ 将来オプション |
| B案 | ◎ | ◎ | ✅ 採用 |

## Step 5: 論理的整合性をチェックする

### チェックリスト

- [ ] Objectiveは「自社視点」ではなく「顧客視点」で書かれているか？
- [ ] WhoとObjectiveの接続が説明できるか？
- [ ] インサイトが具体的に言語化されているか？
- [ ] WhatはインサイトをどうWhyつくのか説明できるか？
- [ ] HowはWhatを実現する手段として適切か？
- [ ] 判断ポイントを満たしているか確認したか？

## Step 6: 曖昧語を定義に落とす

| 曖昧語 | 定義 |
|---|---|
| 「価値を産む」 | ターゲットの具体業務Xの単位あたりコストを減らし、事業KPIに連鎖する変化を起こすこと |
| 「課題を解決する」 | 業務フロー上の"摩擦"を特定し、再現性をもって除去した状態を作ること |
| 「成長する」 | 同じ時間・同じ人数で、より大きな事業インパクトを出せる状態になること |

## アウトプット構成（推奨）

```
# プロジェクト名：戦略整理

## 背景
## Objective
## Who（戦略ターゲット / コアターゲット / インサイト）
## What（MVP / 成功判断）
## How（判断ポイント / シナリオ選択 / 体験設計）
## 成功指標（Lead / Lag）
## アクションプラン（Phase 1, 2, 3...）
## 将来検討事項
```
