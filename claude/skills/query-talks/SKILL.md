# query-talks: 議事録検索

`transcript_markdown/` および `meetings/` 内の議事録を自然言語クエリで検索し、該当ファイルのパスを返す。

## トリガー

- `/query-talks <クエリ>` で明示的に呼び出す
- 「議事録探して」「〇〇の会議どこ」「あの会議で何言ってた」
- 「昨日の1on1」「先週のDXU朝会」「〇〇さんがXXについて何て言ってたか」

## 入力

`$ARGUMENTS` = 自然言語クエリ

例:
- `昨日の河野さん1on1`
- `先週のDXU朝会`
- `佐藤さんがDifyについて何て言ってたか`
- `3月の採用関連の会議`

## 検索ルート判定

クエリを解析し、検索ルートを選択する:

| ルート | 判定基準 | 例 |
|--------|---------|-----|
| **Route A: ファイル名マッチ優先** | 人名なし。日付 + 会議種別/部門コードで絞れる | 「先週の朝会」「昨日のDXU定例」 |
| **Route B: 内容検索優先** | 人名あり、またはトピックのみ | 「河野さんの1on1」「静馬さんの朝会」「Difyの話」 |

**人名がファイル名に含まれないため、人名クエリは必ず Route B を使う。**

## 実行手順

### Step 1: クエリ解析

クエリから以下を抽出する:

1. **日付表現** → YYYYMMDD に変換（変換ルールは `references/search-protocol.md` の「日付解決ルール」を参照）
2. **人名** → 姓のみ抽出（「河野さん」→「河野」）
3. **会議種別** → 1on1, 朝会, 定例, 共有会, キックオフ 等
4. **トピックキーワード** → Dify, 採用, リスク 等
5. **部門コード** → DXU→dx, 開発→dev, 人事→hrd, エンジニアリング→eng, 組織→org

### Step 2: 検索実行

検索アルゴリズムの詳細は `references/search-protocol.md` に従う。

**Route A（ファイル名マッチ優先）**: 人名なしのクエリ
- 月プレフィックス Glob で transcript_markdown/ + meetings/ を検索 → 日付フィルタ → ファイル名キーワードフィルタ

**Route B（内容検索優先）**: 人名ありのクエリ、またはトピックのみ
- 月プレフィックス Glob でファイル一覧取得 → Grep で人名/トピックを内容検索 → 突合

### Step 3: 結果出力

以下のフォーマットで返す:

```markdown
## 検索結果

**クエリ**: {元のクエリ}
**解釈**: 日付={解決した日付}, 人名={人名}, 種別={会議種別}

### ヒット（N件）

1. `transcript_markdown/{ファイル名}`
2. `transcript_markdown/DXU/{ファイル名}`
3. `meetings/mcdx/{ファイル名}`
...
```

**出力ルール**:
- ファイルパスを返す（内容の要約は返さない）
- 最大10件
- 日付降順ソート（新しい順）
- 各ファイルにマッチ根拠を1行添える（「ファイル名: 河野, 1on1」「内容: 出席者に河野を確認」等）
- ヒット0件の場合: キーワードを緩めた再検索を1回試みる。それでも0件なら「該当する議事録が見つかりませんでした」と返す

## 他スキルからの利用

このスキルの検索ロジックは `references/search-protocol.md` に独立して記述されている。
他スキル（task-assign, 1on1-prep 等）が議事録を検索する際は、search-protocol.md を参照すること。

```
議事録の検索方法は .claude/skills/query-talks/references/search-protocol.md を参照。
```
