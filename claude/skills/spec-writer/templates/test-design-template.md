> 差分仕様の場合、本テスト設計書はベース仕様からの**差分のみ**を対象とします。

## 概要

本ドキュメントは、要件定義書の各Acceptance Criteriaに対して、どのテストレベル（単体・統合・E2E）でカバーするかを定義します。

## テストレベル定義

| テストレベル | 略称 | 説明 | ツール |
|-------------|------|------|--------|
| 単体テスト | Unit | 個々のクラス・関数を独立してテスト。外部依存はモック化 | pytest, unittest.mock |
| 統合テスト | Integ | 複数コンポーネントの連携をローカルでテスト（外部サービス実接続含む）。デプロイ不要 | pytest |
| E2Eテスト | E2E | デプロイ後に実行。Lambda/Step Functions を含む全体フローを実環境で検証 | AWS CLI / Terraform test / 手動確認 |
