# 良い提案の例

reflectの提案品質を安定させるためのサンプル集。

## スキル化の良い例

### 例1: 定型ワークフローの検出

```
### 提案1: スキル化 - /sync-competitors

**概要**: アイデアの競合調査ファイルから新規競合を抽出し、競合調査CSVに追加

**検出理由**: セッション中に3回、同じ手順（競合ファイル読み→CSV読み→差分抽出→CSV追記）を繰り返した

**ワークフロー**:
1. 指定されたアイデアの競合調査ファイルを読む
2. 既存の競合調査CSVを読む
3. 未登録の競合を抽出
4. CSVに追記

**保存先**: `.claude/skills/sync-competitors/SKILL.md`
```

**良いポイント**: 検出理由が具体的（3回繰り返した）、ワークフローが明確

### 例2: テンプレート的処理の検出

```
### 提案2: スキル化 - /to-press

**概要**: ソリューション仮説をプレスリリース形式に変換

**検出理由**: ソリューション仮説を別フォーマットに変換する作業で、入力→変換ルール→出力が毎回同じパターン

**保存先**: `.claude/skills/to-press/SKILL.md`
```

## ルール追加の良い例

### 例1: ユーザーの修正指示からの検出

```
### 提案3: ルール追加 - コミットメッセージは日本語で

**検出理由**: ユーザーが2回、英語のコミットメッセージを日本語に直すよう指示した

**内容**:
コミットメッセージは日本語で書く。英語は使わない。

**保存先**: `.claude/rules/コミットメッセージは日本語.md`
```

### 例2: 手戻りからの教訓

```
### 提案4: ルール追加 - ファイル削除前に参照チェック

**検出理由**: スキルファイルを削除した後、他のスキルから参照されていたことが判明し手戻りが発生した

**内容**:
ファイルを削除する前に、そのファイルを参照している箇所がないかGrepで確認する。

**保存先**: `.claude/rules/削除前に参照チェック.md`
```

## CLAUDE.md更新の良い例

```
### 提案5: CLAUDE.md更新 - 新ディレクトリの追記

**検出理由**: `ideas/persona/` ディレクトリが新規作成されたが、CLAUDE.mdのディレクトリ構成に未反映

**変更内容**:
ディレクトリ構成セクションに `persona/` を追加

**保存先**: `CLAUDE.md` ディレクトリ構成セクション
```

## 悪い提案の例（避けるべき）

- **曖昧**: 「コードをきれいにする」→ 具体的に何をどうするか不明
- **一回きり**: 「このバグを修正する」→ 定型パターンではない
- **既存と重複**: 既にルールやスキルに存在する内容の提案
- **過剰**: 1セッションで5件以上の提案 → 本当に必要なものだけに絞る
