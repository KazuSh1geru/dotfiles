# Reflect

セッションで行った作業・やりとりを分析し、リポジトリの知識として蓄積する提案を行う。

## 実行モード

### 同セッション（会話コンテキストあり）
現在の会話コンテキストをそのまま分析する。

### 別セッション（前回セッションの振り返り）
1. プロジェクトディレクトリのハッシュを算出: `echo -n "$PWD" | md5`
2. `/tmp/.claude-last-session-{project_hash}.json` からtranscript_pathを取得
3. transcriptファイルを読み込んで分析

## 分析の3観点

### 1. スキル化
繰り返し作業や定型ワークフローをスキルとして切り出す。

**検出パターン**:
- 同じ手順を複数回実行している
- 決まった順序で複数のツールを使っている
- 入力を受けて定形の出力を生成している

**保存先**: `.claude/skills/[skill-name]/SKILL.md`

### 2. ルール追加
ユーザーの好みや暗黙の前提をルールとして明文化する。

**検出パターン**:
- ユーザーが修正を指示した箇所（＝暗黙のルール違反）
- 明示的に「こうして」「こっちがいい」と好みを示した箇所
- 失敗して手戻りした箇所（＝教訓）
- 繰り返し出てくる前提条件

**保存先**: `.claude/rules/[ルール名].md`
- 1ルール1ファイル、簡潔に記述
- ファイル名は内容を表す日本語
- 特定ファイルにのみ適用すべきルールは `paths` frontmatterでスコープを絞る

### 3. CLAUDE.md更新
新しいワークフローやディレクトリ構成変更など、プロジェクト全体に影響する知識。

**検出パターン**:
- 新しいディレクトリが作られた
- 新しいワークフローが確立された
- 命名規則が追加・変更された

**保存先**: `CLAUDE.md` の該当セクション

## Workflow

1. **コンテキスト取得**
   - 同セッション: 現在の会話から分析
   - 別セッション: transcript読み込み

2. **重複チェック**（[references/analysis-sources.md](references/analysis-sources.md) 参照）
   - `.claude/skills/` の既存スキル一覧を確認
   - `.claude/rules/` の既存ルール一覧を確認
   - `CLAUDE.md` の現在の内容を確認
   - 既存と重複する提案は除外
   - 保存先の判断は analysis-sources.md の「保存先の使い分け」に従う

3. **分析結果を表形式で出力**（[references/good-examples.md](references/good-examples.md) の品質基準に従う）

```
## セッション振り返り結果

| # | 種別 | 内容 | 保存先 |
|---|------|------|--------|
| 1 | スキル化 | [概要] | `.claude/skills/[name]/` |
| 2 | ルール追加 | [概要] | `.claude/rules/[名前].md` |
| 3 | CLAUDE.md更新 | [概要] | `CLAUDE.md` [セクション名] |

---

### 提案1: [種別] - [タイトル]
[詳細な説明]
[具体的な変更内容のプレビュー]

---

### 提案2: ...

---

適用する提案を番号で選んでね（複数可、例: 1,3）
```

4. **ユーザー選択 → 適用**（[references/skill-template.md](references/skill-template.md) 参照）
   - 選択された提案のみ実行
   - **スキル化提案の場合**:
     1. reflectが検出した情報（概要・検出理由・ワークフロー案）をコンテキストとしてまとめる
     2. `/skill-creator` を呼び出し、コンテキストを渡してスキル作成を委譲する
     3. skill-creatorのインタビュー→生成→テストフローに従う
     4. skill-creator完了後、`CLAUDE.md` の「利用可能なスキル」と `README.md` を更新
   - **ルール追加の場合**: skill-template.md のルールファイルフォーマットに準拠して直接作成（全ファイル適用か `paths` スコープかを必ず判断）
   - **CLAUDE.md更新の場合**: 該当セクションを直接編集

5. **コミット**
   - 適用した変更をコミット（コミットルールに従う）

## 提案がない場合

分析した結果、蓄積すべき知識がない場合は正直に「特に蓄積すべきものはなかったよ〜」と伝える。無理に提案を作らない。

## Validation

- [ ] 既存スキル・ルール・CLAUDE.mdとの重複がないこと
- [ ] 各提案が具体的で実行可能であること
- [ ] ユーザーの選択なしに変更を適用しないこと
- [ ] アプリ固有のルールには `paths` frontmatterが付いていること（例: `paths: ["apps/pain-catalog/**"]`）
