# スコアカード v1（research）

リサーチレポートの品質を判定するための評価基準。
Generator が出力したリサーチレポートに対し、Reviewer（coverage-reviewer・critical-analyst）がこのスコアカードで判定する。

全リサーチタイプ（competitor / technology / market / general）で共通。

## MUST（全通過必須）

すべて PASS でなければ収束しない。1つでも FAIL なら改善ループを継続する。

| ID | 項目 | 判定基準 | FAIL の典型例 |
|---|---|---|---|
| M1 | 調査対象と目的が冒頭で明確 | 何を・なぜ調べたかが冒頭で明示されている。読者がこのレポートを読む理由がわかる | 目的の記述がない。調査範囲が不明確 |
| M2 | 情報源が明記されている | 各主張・データに情報源（URL、文献名、データセット名）が付記されている。出典なき主張が存在しない | 「〜と言われている」「一般的に〜」で出典なし |
| M3 | 直接的な事実と分析が分離されている | 「調査で判明した事実」と「事実に基づく分析・考察」が明確に区別されている | 事実と推測が混在し、どこまでが調査結果でどこからが考察か不明 |
| M4 | 調査の限界・未調査領域が明記されている | 「調べられなかったこと」「情報が不足している領域」が明示されている | 調査が万全であるかのように書かれ、盲点が無視されている |
| M5 | アクション示唆または結論が明確 | 調査結果から導かれる「次に何をすべきか」または「何が言えるか」が明確 | 情報の羅列で終わり、So What が不在 |

## WANT（改善推奨）

MUST を全通過した上で、品質向上のために判定する。

| ID | 項目 | 判定基準 | YES の例 |
|---|---|---|---|
| W1 | 競合/代替手段の網羅性 | 直接競合だけでなく間接競合・代替手段・DIY解決策も調査されている | 「ユーザーが現在Excelで対応している」等の代替手段にも言及 |
| W2 | 定量データの活用 | 市場規模、ユーザー数、レビュー件数等の定量データで裏付けられている | 「ITreviewで425件のレビュー」「市場規模は〇〇億円」 |
| W3 | 比較マトリクスの存在 | 調査対象を横断的に比較できる表・マトリクスが含まれている | 機能比較表、価格比較表、SWOT 比較 |
| W4 | ユーザー視点の情報 | レビュー・口コミ・ユーザーの声など、実際の利用者の視点が含まれている | 「ユーザーからは〇〇が高評価」「〇〇に不満の声」 |
| W5 | 時系列・トレンド情報 | 現在のスナップショットだけでなく、変化の方向性・トレンドに言及している | 「直近1年で〇〇にシフト」「2024年に〇〇が登場」 |

## 収束条件

```
IF MUST 全項目 PASS:
  IF ループ回数 >= 2 OR WANT の YES が 3/5 以上:
    → 完了。最終リサーチレポートを出力
  ELSE:
    → WANT 改善のためもう1ループ
ELSE:
  IF ループ回数 >= 5:
    → 打ち切り。残存課題レポートを人間に提示
  ELSE:
    → MUST FAIL 項目の改善指示を出して次ループ
```

| M3 | PASS/FAIL | （具体的な根拠） | （該当箇所） |
| M4 | PASS/FAIL | （具体的な根拠） | （該当箇所） |
| M5 | PASS/FAIL | （具体的な根拠） | （該当箇所） |
